# Memory System

## How Memory Loads

Claude Code auto-loads `memory/MEMORY.md` every session via a symlink:
```
~/.claude/projects/-home-connor-AI-Tools/memory/MEMORY.md
  → /home/connor/AI-Tools/memory/MEMORY.md
```

The symlink was created during the rebuild (2026-03-03). This means any edit to
`memory/MEMORY.md` is immediately reflected in the next Claude session.

## Memory Files

| File | Purpose |
|------|---------|
| `memory/MEMORY.md` | Primary snapshot (<200 lines) — auto-loaded |
| `memory/sessions/YYYY-MM-DD.md` | Per-session logs (auto-generated by hooks) |
| `memory/decisions.md` | Architectural decisions log |
| `memory/patterns.md` | Code patterns discovered |

## How to Update Memory

### Add a quick fact
```bash
tools/memory add "LiteLLM v1.82 released, tested OK"
```

### Run end-of-session sync
```bash
tools/sync
```
This: logs session timestamp → syncs beads → commits memory/ if changed.

### Edit directly
```bash
$EDITOR memory/MEMORY.md
```
Keep it under 200 lines (Claude Code truncates after that).

## Automatic Session Logging

The Claude Code `Stop` hook fires `hooks/session-end.sh` after each response:
1. Appends `HH:MM session end` to `memory/sessions/YYYY-MM-DD.md`
2. Runs `bd sync --flush-only` silently
3. Commits session log if changed

This is async (non-blocking, <10s timeout) so it doesn't slow down interactions.

## RAG Search

For searching project history beyond what fits in MEMORY.md:
```bash
python3 rag/knowledge_base.py search "LiteLLM setup"
python3 rag/knowledge_base.py stats
python3 rag/knowledge_base.py reindex  # after adding new docs
```

ChromaDB stores in `~/.local/share/ai-knowledge/`.
